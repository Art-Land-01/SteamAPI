# SteamAPI

Небольшое консольное Java-приложение, которое получает данные пользователя
Steam через [Steam Web API](https://steamcommunity.com/dev) — профиль,
список игр и достижения — и сохраняет их в базу данных PostgreSQL.

Можно посмотреть свою статистику по играм и достижениям, а также текущий
статус пользователя (онлайн/офлайн, во что играет, когда был последний раз
в сети).

## Возможности

- Получение публичного профиля игрока (имя, аватар, статус, последний визит).
- Список игр пользователя.
- Поиск игры по (части) названия
(пока что лучше сохранять оригинальную офрографию - пробелы и апострофы особенно).
- Отслеживание выбранных игр и сохранение их в базу.
- Получение и сохранение достижений по конкретной игре, подсчёт количества полученных.

## Стек

- Java 22, Maven
- Apache HttpClient (HTTP-запросы), Jackson (разбор JSON)
- PostgreSQL + HikariCP (пул соединений)

## Архитектура

```
SteamAPI    -> обращается к Steam Web API, возвращает «сырой» JSON
Parser      -> превращает JSON в объекты модели
WorkerBee   -> связывает API, парсер и репозитории
*Repo -> работа с БД (пользователи, отслеживаемые игры, достижения)
Database    -> пул соединений HikariCP
Game / PlayerStat / GameAchievements -> модели данных
```

## Запуск

1. **База данных** — создайте базу PostgreSQL и таблицы `users`,
   `tracked_games`, `achievements` (под steam_id, app_id и т.п. — см. SQL
   в репозиториях `*Repo`/`*Repository`).

2. **Конфигурация** — скопируйте пример конфига и заполните своими данными:
   ```bash
   cp src/main/resources/config.properties.example src/main/resources/config.properties
   ```
   Укажите свой Steam Web API key (получить тут:
   https://steamcommunity.com/dev/apikey) и параметры подключения к БД.
   `config.properties` уже в `.gitignore`, но в любом случае APIKEY и steamID лучше не выкладывать в открытый доступ - просто совет).

3. **Сборка:**
   ```bash
   mvn clean package
   ```

4. **Запуск:**
   ```bash
   mvn compile exec:java -Dexec.mainClass=com.practice.Main
   ```
